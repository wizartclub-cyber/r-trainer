# HANDOFF — Тренажер «Р» / R Trainer

Технічний опис проєкту для того, хто продовжить розробку. Стан на 2026-07-10.

## 1. Що це

Односторінковий вебзастосунок-тренажер для постановки звуку **«Р»**: 56-денний курс
із трьох фаз. Немає збірки, немає бекенду — один файл `index.html` + допоміжні файли PWA.
Трилінгвальний: 🇺🇦 українська, 🇸🇰 slovenčina, 🇬🇧 English.

- **Live:** <https://wizartclub-cyber.github.io/r-trainer/>
- **Репозиторій:** <https://github.com/wizartclub-cyber/r-trainer> (public)
- **GitHub-акаунт:** `wizartclub-cyber`, git по SSH (`git@github.com:wizartclub-cyber/r-trainer.git`)
- **Хостинг:** GitHub Pages, гілка `main`, корінь `/`, HTTPS. Деплой = `git push` (Pages збирається сам ~1–2 хв).

## 3. Архітектура `index.html`

Три шари, усі в одному файлі:

1. **`I18N`** — об'єкт із трьома мовами (`uk`, `sk`, `en`). Кожна мова містить:
   - рядки інтерфейсу (`tabToday`, `saved`, `phases`, …);
   - `EX` — вправи, згруповані за блоками: `gym`, `vibr`, `auto1`, `auto2`.
   - `L()` повертає поточну мову (`I18N[state.lang]`).

2. **Стан** — `state = { start, lang, days }`, `curDay` (поточний день, що показується).
   - `state.days[n] = { done:{exId:true}, vib:'yes'|'partial'|'no'|null, notes, prog:{exId:count} }`.

3. **Рендер** — `renderToday()`, `renderProgress()`, `renderCtrl()`, `renderRec()`;
   перемикання `setView()`; мова `applyLang()` + `setLang()`.

### Модель курсу
- 56 днів = 8 тижнів. `phaseOfWeek(w)`: тижні 1–2 → фаза 1 (`gym`), 3–4 → фаза 2 (`vibr`), 5–8 → фаза 3.
- Фаза 3: тижні 5–6 → `auto1`, 7–8 → `auto2`. Див. `exercisesFor(week)`.

### Типи вправ (`ex.type`)
- `reps` — лічильник (`target`, `unit`); кнопка «+1», авто-виконано при досягненні target.
- `timer` — таймер-кільце (`sets`, `sec`, `unit`); авто-інкремент підходу по завершенню.
- `check` — просто галочка.

### i18n — головний принцип
**Ідентифікатори вправ (`id`) спільні для всіх мов.** Прогрес зберігається за `id`,
тому перемикання мови нічого не губить. Перекладається лише текст (`name`, `desc`, `unit`) — не структура.

`applyLang()` проставляє **статичні** тексти DOM (через `id` елементів);
`renderToday`/`renderProgress` читають `L()` для **динамічних**. При зміні мови
`setLang()` викликає `applyLang()` + повторний рендер поточного екрана.

## 4. Сховище

- **Прогрес:** `localStorage`, ключ `rtrainer:v1`, значення — JSON усього `state`.
  Запис через `persist()` (debounce 300 мс). Читання — `load()`.
- **Записи голосу:** IndexedDB, база `rtrainer`, стор `rec`. Ключ = номер дня (число),
  значення = аудіо-`Blob`. Обгортка: `idb()`, `idbPut/idbGet/idbDel/idbKeys`.
  Один (останній) запис на день; «Ще раз» перезаписує, 🗑 видаляє.
- **Експорт/імпорт** (`exportData`/`importData`) — лише `state` (прогрес), **без аудіо**
  (блоби завеликі для 5 МБ localStorage і роздули б JSON). Аудіо навмисно device-local.

## 5. PWA

- Реєстрація SW наприкінці `index.html` (`navigator.serviceWorker.register('sw.js')`), помилка безпечна.
- `sw.js`: `install` кешує app-shell (`.`, `index.html`, `manifest.json`, `icon.svg`),
  `fetch` — cache-first + докешовує успішні GET (так офлайн підхоплює Google Fonts після першого онлайну).
- **ВАЖЛИВО при релізі:** якщо змінюєш `index.html`/ассети — **підніми версію** `const CACHE`
  у `sw.js` (`r-trainer-v4` → `v5`). Інакше встановлені PWA віддаватимуть старий кеш.
  Це єдиний ручний крок деплою, який легко забути.

## 6. Як робити типові зміни

**Додати мову** (напр. `pl`):
1. Додай `I18N.pl = { … }` — скопіюй `en` як каркас, переклади всі рядки і `EX` (ті самі `id`!).
2. Додай `<option value="pl">…</option>` у `#langSel`.
3. Все. `L()`, `applyLang()`, `setLang()` підхоплять автоматично.

**Змінити вправу / дозу:** редагуй відповідний запис у `I18N[*].EX` (у всіх мовах, бо `id` спільний).

**Змінити тривалість фаз / курсу:** `phaseOfWeek()`, `exercisesFor()`, і межі `56`/`8` у рендері.

**Задеплоїти:** `git commit` + `git push origin main`. Не забудь `CACHE` у `sw.js` (§5).

## 7. Гочі / нетривіальні рішення

- **`window.storage` → `localStorage`.** Оригінал (`trenazher_R.html`) зберігав через `window.storage`
  (API пісочниці, якого немає у звичайному браузері) — на Pages збереження мовчки не працювало.
  Замінено на `localStorage`. Оригінал лишено локально, але в git **не** доданий (`.gitignore`).
- **Аудіо не в експорті** — свідомо (див. §4).
- **Скасування запису при навігації:** `abortLive()` знімає `mr.onstop` **перед** `stop()`,
  щоб перехід між днями не зберіг незакінчений запис у чужий день.
- **`vib` зберігається як `yes/partial/no`** (не як локалізований текст) — щоб CSS-класи
  (`.sel-yes` тощо) і логіка попередження працювали незалежно від мови.
- **Скоромовки/слова адаптовані під мову**, а не перекладені дослівно (у SK є справжнє
  розкотисте «r»: `Strč prst skrz krk`; для EN — класичні трелі).
- **Попередження про вуздечку:** якщо у фазі вібрації (дні 15–28) ≥8 залогованих днів і всі `vib==='no'`,
  показується порада перевірити під'язикову вуздечку в логопеда (`#vibWarn`).

## 8. Тестування

Браузерного E2E немає (в оточенні немає Playwright/Puppeteer). Перед пушем ганяю в Node:
- синтаксис inline-JS (`vm.Script`);
- усі `getElementById` мають відповідний `id` у розмітці;
- паритет i18n-ключів між uk/sk/en (жодного пропущеного перекладу);
- юніт-тест `recExt()` (вибір розширення файлу);
- round-trip IndexedDB-обгортки на in-memory моку (`put→get→del`, порядок `put(value, key)`).

Не покрито (потребує реального пристрою): сам захват мікрофона `getUserMedia`/`MediaRecorder`.
