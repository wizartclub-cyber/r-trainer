## 0. TL;DR для виконавця

1. Один набір токенів (розділ 2) — усі продукти.
2. Один шрифт: системний стек (SF Pro / Inter fallback).
3. Один акцент: `#0071e3`. Ніяких других акцентів без правки цього ТЗ.
4. Радіус панелей — 8px. Завжди.
5. Ніяких декоративних градієнтів як основного контенту.
6. Різниця між продуктами — тільки в **щільності** та **розмірі типографіки** (розділ 5), не в кольорах і не в компонентах.

## 1. Принципи

| # | Принцип | Що це означає на практиці |
|---|---------|---------------------------|
| 1 | **Спокій** | Білі й майже білі поверхні, м'яка глибина. Колір — лише для дії та статусу, ніколи для декору. |
| 2 | **Точність** | Все вирівняно по сітці 4px. Числа — табличними цифрами. Дати й валюти — за одним форматом. |
| 3 | **Тиша за замовчуванням** | Елемент кричить тільки коли він активний або потребує уваги. Навігація, іконки, бордери — приглушені. |
| 4 | **Щільність без шуму** | В операційних екранах багато даних — але простір між групами робить їх читабельними. |
| 5 | **Контент — герой** | В галереї та на лендінгах UI відступає: макс. білого простору, мінімум хрому. |
| 6 | **Один рух** | Анімації — короткі, ease-out, ніколи не bounce. Рух пояснює, а не розважає. |

**Антипатерни (заборонено):**
- Градієнтні кнопки, неон, glassmorphism поверх контенту.
- Більше 2 шрифтів на екрані.
- Кастомні кольори поза токенами.
- Тіні глибші за `--shadow-lg`.
- Emoji як іконки в продакшн-UI.
- Радіуси 12/16/20px «бо гарніше».

## 2. Токени (ядро системи)

### 2.1 Колір

| Токен | HEX / значення | Використання |
|---|---|---|
| `--bg` | `#f5f5f7` | Фон застосунку/сторінки |
| `--surface` | `#ffffff` | Картки, панелі, таблиці |
| `--surface-translucent` | `rgba(255,255,255,0.72)` + `backdrop-filter: blur(20px)` | Сайдбар, sticky-хедер |
| `--surface-sunken` | `#fafafa` | Заголовки таблиць, disabled-поля |
| `--text` | `#1d1d1f` | Основний текст, заголовки |
| `--text-secondary` | `#6e6e73` | Підписи, метадані, плейсхолдери |
| `--text-tertiary` | `#86868b` | Дизейблений текст, hint (**не для змістовного тексту**) |
| `--line` | `rgba(0,0,0,0.10)` | Бордери, роздільники |
| `--line-strong` | `rgba(0,0,0,0.18)` | Бордери інпутів |
| `--accent` | `#0071e3` | Фон primary-кнопки, активний стан, іконки |
| `--accent-text` | `#006ada` | **Синій текст і посилання** на `--bg` (див. правило нижче) |
| `--accent-hover` | `#0077ed` | Hover primary |
| `--accent-active` | `#006edb` | Pressed |
| `--accent-soft` | `rgba(0,113,227,0.08)` | Фон активного пункту меню, виділений рядок |
| `--success` | `#12805c` | Оплачено, підтверджено, продано |
| `--success-soft` | `rgba(18,128,92,0.10)` | Фон бейджа |
| `--warning` | `#8a5a00` | Очікує, прострочення < 7 днів |
| `--warning-soft` | `rgba(138,90,0,0.10)` | Фон бейджа |
| `--danger` | `#b3261e` | Помилка, скасування, видалення |
| `--danger-soft` | `rgba(179,38,30,0.10)` | Фон бейджа |
| `--focus-ring` | `0 0 0 4px rgba(0,113,227,0.25)` | Кільце фокуса |

> **Правило контрасту (перевірено розрахунком WCAG):**
> - `--text` на `--bg` = **15.5:1**, на `--surface` = 16.8:1 ✅
> - `--text-secondary` = 4.7 / 5.1:1 ✅
> - `--success` 4.9:1 · `--warning` 5.9:1 · `--danger` 6.5:1 на білому ✅ — саме тому вони затемнені. **Не замінювати на «яскраві» варіанти.**
> - Білий текст на `--accent` = 4.7:1 ✅ → кнопка ок.
> - ⚠️ `--accent` (#0071e3) **як текст на сірому `--bg`** = 4.31:1 → **не проходить AA**. Для синього тексту/посилань на сірому фоні використовуй `--accent-text` (#006ada, 4.73:1). На білому `--surface` `--accent` можна як текст (4.7:1).
> - ⚠️ `--text-tertiary` = 3.6:1 → лише для disabled-стану й декору (WCAG виключає disabled). **Ніколи не для змістовного тексту.**

Числа в таблицях і сумах: `font-variant-numeric: tabular-nums;` — обов'язково.

| H1 (застосунок) | 28 / 1.2 | 600 | −0.01em |
| H2 | 22 / 1.3 | 600 | −0.01em |
| H3 | 17 / 1.4 | 600 | 0 |
| Body | 15 / 1.5 | 400 | 0 |
| Body small / таблиці | 13 / 1.45 | 400 | 0 |
| Caption / label | 12 / 1.35 | 500 | +0.01em |
| Button | 15 / 1 | 500 | 0 |

Максимальна ширина суцільного тексту — **72ch** (лендінг/документи).

### 2.3 Простір, радіус, тінь, рух

```
--space: 4 · 8 · 12 · 16 · 20 · 24 · 32 · 40 · 56 · 80  (px, сітка 4px)

--radius-sm: 6px    (бейджі, дрібні контроли)
--radius:    8px    (панелі, картки, інпути, кнопки — ДЕФОЛТ)
--radius-lg: 12px   (модалки, великі медіа-картки)
--radius-full: 999px (аватари, пілюлі)

--shadow-sm: 0 1px 2px rgba(0,0,0,0.04);
--shadow:    0 2px 8px rgba(0,0,0,0.06);
--shadow-lg: 0 12px 32px rgba(0,0,0,0.12);   ← максимум, глибше не буває

--ease: cubic-bezier(0.4, 0, 0.2, 1);
```

## 4. Компоненти

Кожен компонент має 5 станів: **default → hover → active → focus → disabled**. Плюс `loading` і `error` там, де застосовно. Компонент без описаних станів не приймається.

### 4.1 Кнопки

| Варіант | Фон | Текст | Бордер | Використання |
|---|---|---|---|---|
| **Primary** | `--accent` | `#fff` | — | Одна на екран. Головна дія. |
| **Secondary** | `--surface` | `--text` | `1px --line-strong` | Другорядні дії |
| **Ghost** | прозорий | `--accent` | — | Дії в рядках таблиць, утиліти |
| **Danger** | `--surface` | `--danger` | `1px --danger` | Видалення, скасування. Завжди з підтвердженням. |

Розміри: `sm` 32px / `md` 40px (дефолт) / `lg` 48px (лендінг CTA).
Padding: 0 16px (sm/md), 0 24px (lg). Радіус 8px. Вага тексту 500.
Hover: primary → `--accent-hover`; secondary/ghost → фон `--accent-soft`.
Disabled: `opacity: 0.4; cursor: not-allowed;` — колір не змінюємо.
Loading: спінер 16px замість тексту, ширина кнопки фіксується.

### 4.2 Поля вводу

- Висота **40px**, padding 0 12px, радіус 8px.
- Border `1px var(--line-strong)`, фон `--surface`.
- Focus: border `--accent` + `--focus-ring`.
- Error: border `--danger`, повідомлення 12px `--danger` під полем. Помилка з'являється на blur, не під час набору.
- Label — **завжди зверху** (12px, 500, `--text-secondary`), ніяких плейсхолдер-лейблів.
- Обов'язкове поле: `*` кольору `--danger` після лейбла.
- Textarea: min-height 96px, resize vertical.
- Select — нативний або кастомний з тим самим 40px-боксом.

### 4.3 Панелі / картки

```
background: var(--surface);
border: 1px solid var(--line);
border-radius: 8px;
box-shadow: var(--shadow-sm);
padding: 20px;  /* 16px в щільних ERP-екранах */
```
Hover для клікабельних карток: `box-shadow: var(--shadow)` + `transform: translateY(-1px)` за `--dur-fast`.
Заголовок картки: 17/600. Під ним — 4px, далі метадані 13px `--text-secondary`.

### 4.4 Таблиці

- Рядок: висота **44px** (компакт 36px). Вертикальний padding 12px.
- Хедер: фон `--surface-sunken`, текст 12px/500/`--text-secondary`, uppercase **не** використовуємо.
- Роздільники: `1px --line` тільки горизонтальні. Вертикальних ліній немає.
- Zebra-striping — **ні**. Розділяємо лініями.
- Hover рядка: фон `--accent-soft`.
- Виділений рядок: фон `--accent-soft` + ліва межа 2px `--accent`.
- Числові колонки — вирівняні праворуч, tabular-nums.
- Sticky header при скролі. Дії рядка — ghost-кнопки, видимі на hover, але доступні з клавіатури завжди.
- Пусто → empty state: іконка 32px, заголовок 15/500, підказка 13px, одна primary-кнопка.

### 4.5 Навігація (сайдбар)

- Ширина 260px, фон `--surface-translucent` + blur(20px), права межа `1px --line`.
- Пункт: висота 36px, радіус 6px, іконка 18px + текст 14px.
- Default: іконка й текст `--text-secondary`.
- Hover: фон `rgba(0,0,0,0.04)`.
- Active: фон `--accent-soft`, іконка й текст `--accent`, вага 500.
- Групи розділені 20px + заголовок групи 11px/600/`--text-tertiary`.
- Згорнутий стан 64px: тільки іконки + tooltip.

### 4.6 Бейджі / статуси

Висота 22px, padding 0 8px, радіус 6px, текст 12px/500.
`Paid/Confirmed/Sold` → `--success` на `--success-soft`.
`Pending/Awaiting` → `--warning` на `--warning-soft`.
`Cancelled/Failed/Overdue` → `--danger` на `--danger-soft`.
`Draft/Neutral` → `--text-secondary` на `rgba(0,0,0,0.05)`.
Крапка-індикатор 6px опційно ліворуч. **Статус ніколи не передається лише кольором** — завжди є текст.

### 4.7 Модалки

Ширина 480px (sm) / 640px (md) / 880px (lg). Радіус 12px, `--shadow-lg`.
Оверлей: `rgba(0,0,0,0.32)` + blur(4px).
Поява: opacity 0→1 + scale 0.98→1 за `--dur-slow`.
Футер справа: Secondary (Скасувати) + Primary. Esc і клік по оверлею закривають — крім форм з незбереженими змінами (тоді підтвердження).

### 4.8 Тости

Темний матеріал: `rgba(29,29,31,0.88)` + blur(20px), текст білий 14px, радіус 8px, `--shadow-lg`.
Позиція: bottom-center (застосунок) / top-right (сайти). Автозакриття 4с, помилки — не закриваються самі.

### 4.9 Іконки

Один набір: **Lucide** (stroke 1.5px), розміри 16/18/20/24. Колір успадковується від тексту. Своїх іконок не малюємо.

## 5. Профілі продуктів

Токени однакові. Змінюються тільки щільність, розмір типографіки й акценти макета.

### 5.1 ERP / CRM / дашборди (внутрішні системи)

**Мета:** максимум даних за мінімум кліків. Ефективність > краси.

- Body 14px, таблиці 13px. Щільність: рядок 36–44px.
- Дашборд — **metric-first**: 4 KPI-картки зверху (велике число 32/600 tabular, підпис 12px, дельта зі стрілкою `--success`/`--danger`), нижче — робочі таблиці.
- Ніяких hero-блоків, банерів, маркетингових ілюстрацій.
- Графіки: 1 колір даних (`--accent`), сітка `--line`, без 3D, без легенд якщо серія одна.
- Всі частотні дії доступні з клавіатури. Пошук — `/` або `Cmd+K`.
- Печать/PDF — утилітарні ghost-кнопки біля заголовка редактора, не в футері.
- Мапа: підкладка світла, сітка `--accent` на 8% прозорості, шари-оверлеї легкі.

### 5.2 Маркетингові сайти (оренда, нерухомість)

**Мета:** довіра + конверсія в бронювання/заявку.

- Body 17px, Display 48–64px. Повітря: секції 80–120px вертикально.
- Hero: одне велике фото об'єкта, заголовок ≤ 8 слів, підзаголовок 1 рядок, **одна** primary-кнопка 48px. Ніяких відео-фонів.
- Фото — головний носій довіри: мінімум 1600px по ширині, WebP/AVIF, lazy-load, співвідношення 3:2 або 16:9 (не міксувати на одній сторінці).
- Блок цін і доступності — sticky-картка справа на lg, приліплена внизу на mobile.
- Соцдоказ: рейтинг + кількість відгуків + логотипи платформ у сірому (`--text-tertiary`), не кольорові.
- Форма заявки: макс. 4 поля. Кожне зайве поле = мінус конверсія.
- Мова: SK / EN / UA. Ціни в EUR, формат `1 250 €`. Дати `DD.MM.YYYY`.
- Обов'язково: cookie-банер (GDPR), сторінка політики, юр. дані підприємця у футері.
- Performance-бюджет: LCP < 2.0s, CLS < 0.1, головна < 500KB.

### 5.3 Арт-галерея / портфоліо

**Мета:** робота — герой. UI має зникнути.

- Фон `--surface` (**білий**, не `--bg`) — картина висить на білій стіні.
- Сітка робіт: masonry або 3-колонкова, gutter 24–32px. Без бордерів і тіней у превʼю.
- Ніяких overlay-підписів поверх зображень. Підпис — під роботою: назва (15/500, курсив опційно), автор/техніка/рік/розмір (13px `--text-secondary`), ціна (15/500) або «Sold» бейджем.
- Клік → lightbox: фон `rgba(255,255,255,0.96)` (світлий, не чорний), зображення по центру, метадані знизу малим кеглем, стрілки/Esc/свайп.
- Акцентний блакитний **не з'являється** поверх мистецтва — тільки в CTA («Запит на роботу», «Забронювати перегляд»).
- Зображення: макс. якість, `<img>` з правильним `alt`; водяний знак — не рекомендується (псує сприйняття), замість нього — обмеження роздільності до 1600px.
- Сторінка роботи: зображення ≥ 60% ширини екрана, праворуч — вузька колонка з інфо та однією CTA.

### 5.4 Документи: PDF, звіти, презентації

**Мета:** офер/звіт має виглядати як продовження бренду.

- Друк: фон **білий** (не `#f5f5f7`), текст `#1d1d1f`, лінії `#e5e5e7` (rgba у друці не працює стабільно).
- Шапка: назва компанії 11px/600 ліворуч, дата документа праворуч, лінія знизу.
- Заголовок документа: 24/600, під ним підзаголовок 13px `--text-secondary`.
- Таблиці: ті самі правила — тільки горизонтальні лінії, числа праворуч, tabular-nums.
- Футер: сторінка X/Y, юр. дані, контакт. Кожна сторінка.
- Формат: A4, поля 20mm. Шрифт у PDF — Inter (SF Pro не ліцензований для вбудовування).
- Презентації: 16:9, одна думка на слайд, ≤ 6 рядків тексту, титул 40/600, заголовок слайда 28/600, тіло 18px. Графіки — тільки `--accent` + сірі.
- Валюта/дата — той самий формат, що на сайті. Ніяких `$` у європейських документах.

## 6. Контент і мова інтерфейсу

- Кнопки — дієслово в інфінітиві/наказовій: «Зберегти», «Створити бронювання». Не «OK».
- Помилка = що сталося + що зробити. «Не вдалося зберегти. Перевірте з'єднання і спробуйте ще.»
- Ніяких «Упс!», «Ой», жартів у помилках.
- Дата `DD.MM.YYYY`, час 24h, гроші `1 250,00 €` (EU-формат, нерозривний пробіл перед €).
- Пусті стани пояснюють, що робити далі, і дають кнопку.

## 7. Доступність (обов'язковий мінімум)

- Контраст тексту ≥ 4.5:1, великого тексту (≥ 24px або 19px/600) ≥ 3:1.
- Всі інтерактивні елементи доступні з клавіатури; `:focus-visible` завжди видимий.
- Тач-таргет ≥ 44×44px на мобільних.
- Кожне зображення — `alt`; декоративні — `alt=""`.
- Форми: `<label for>` реальний, не aria-хак.
- Стан ніколи не передається тільки кольором.
- `prefers-reduced-motion` підтримано.
- Мовний атрибут `<html lang>` правильний на кожній локалі.
